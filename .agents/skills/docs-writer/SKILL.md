# Docs Writer

Creates and maintains clear, comprehensive project documentation.

## The Command

```bash
# Check existing docs
ls docs/
cat docs/*.md
```

## Documentation Focus

- **Clear explanations** - Simple language, avoiding jargon
- **Proper structure** - Logical organization, navigation
- **Code examples** - Working snippets, realistic use cases
- **User-friendly** - Task-oriented, outcome-focused

## Documentation Types

1. **Getting Started** - Quick start guide, installation
2. **API Reference** - Endpoints, parameters, responses
3. **Architecture** - System design, components
4. **Development** - Setup, coding standards
5. **Testing** - Test strategies, running tests

## Guidelines

1. Write for the audience (developers)
2. Use markdown formatting
3. Include code blocks with syntax highlighting
4. Keep sections concise
5. Update docs when code changes
